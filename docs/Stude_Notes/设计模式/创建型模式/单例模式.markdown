# 单例模式

确保某一个类只有一个实例对象,而且自行实例化并向整个系统提供这个实例

> 创建在系统中独一无二的对象

> 为了节约系统资源,有事需要确保系统中某个类只有为一一个实例,当这个唯一实例创建成功之后,无法在创建一个同类型的其他对象,所以的操作都只能基于这个唯一实例

**示例**


```python
class TaskManager:

```

## 主要优点

1. 单例模式提供了对唯一实例的受控访问.可以严格控制客户怎样以及何时访问它.
2. 由于在系统中只存在一个对象.因此 可以节约系统资源.
3. 允许可变数目的实例.基于单例模式,可以获得指定个数的实例对象.(多例模式)

## 主要缺点

1. 单例模式没有抽象层,单例类的扩展有很大的困难
2. 单例类的职责过重,一定程度上违反了单一职责原则.(单例类既提供了业务方法,又提供了创建对象的方法,将对象的创建和对象的功能耦合到了一起)
3. 很多面向对象的语言提供了自动垃圾回收技术.如果共享的对象长时间不被利用,系统会认为它是垃圾,自动销毁并回收资源,下次利用又重新实例化,导致共享的单例对象状态的丢失


## 饿汉式单例类

在类被加载时就将自己实例化

优点:

* 无需考虑多线程的问题,可以确保实例的唯一性
* 调用速度和反应时间要优于懒汉式单例

缺点:

* 系统在运行时无论是否需要使用该对象,都会创建.(会导致加载时间延长)

## 懒汉式单例类

在第一次使用时创建

优点:

* 在使用时创建,无需一直占用资源(但是需要处理好多线程的问题)

缺点:

* 需要添加双重检查锁定等机制进行控制.消耗性能
